# Tameem Sankari 'Energy Zero' ASCII Art — Agent Handoff Document

이 문서는 프로젝트의 연속성을 위해 다음 작업을 수행할 AI 에이전트(또는 작업자)에게 현재까지의 시스템 아키텍처, 렌더링 로직, 그리고 목표를 인계하기 위해 작성되었습니다.

## 1. 프로젝트 개요
* **목표:** Tameem Sankari의 "Energy Zero" 스타일 제너러티브 아트를 시스템으로 구축 (밝기에 따른 시각적 획 밀도 매핑, 퓨어 블루 포인트 컬러).
* **완료된 페이즈:** 
  * Phase 1 (수학적 펄린 노이즈 기반 웹 프로토타입 구현 - `ascii_energy.html`)
  * Phase 2 (실제 비디오 소스 픽셀 파싱 및 ASCII 변환 시스템 구축 - `ascii_video.html`)
* **예정된 페이즈:** 
  * Phase 3 (After Effects 스크립팅: 동일한 시각 로직을 AE 텍스트 레이어로 옮기는 ExtendScript 개발)

## 2. 핵심 로직 및 렌더링 아키텍처 (웹 버전 기준)

다음 에이전트가 After Effects(이하 AE)로 로직을 이식할 때 반드시 참고해야 하는 웹(`p5.js`) 기준의 렌더링 명세입니다.

### 2.1 해상도 및 그리드 맵핑
* **셀(Cell) 크기:** 12px 고정
* **동작 방식:** 1920x1080과 같은 스크린을 12px 크기의 그리드로 분할합니다. 각 12px 사각형의 정중앙에 단일 Monospace 속성 문자가 렌더링됩니다.
* **비디오 샘플링 (Phase 2):** 비디오 레이어 전체를 샘플링하면 무거우므로, `p5.Graphics` 오프스크린 캔버스를 생성해 **그리드 셀 개수(예: 160x90)와 1:1로 일치하는 아주 작은 해상도로 비디오를 다운스케일링**하여 픽셀을 읽습니다. 

### 2.2 루미넌스(밝기) 추출 및 대비 곡선
* **CCIR 601 표준 루미넌스 모델 사용:** `luma = (0.299 * R + 0.587 * G + 0.114 * B) / 255.0`
* 밝기 값(`luma`)은 `0.0` ~ `1.0`의 범위를 가집니다.
* **가중치 대비(Contrast) 곡선 적용:** 아날로그 CRT의 먹먹한 노이즈 게이트(빈 공간은 더 확실하게 비우기)를 시뮬레이션하기 위해, `val = luma - 0.1`로 섀도우를 깎아낸 뒤 `val = pow(val, 1.4)` 수준의 곡선을 통과시켜 최종 렌더링 `val` 값을 추출합니다.

### 2.3 ASCII 문자 세트 대응 (Character Map)
* **모드 A (Block):** `' .:░▒▓█'`
* **모드 B (Text):** `' .YRATONEWE'`
* **매핑 로직:** `val (0.0~1.0)`을 문자열 배열의 길이에 비례하게 매핑하여 인덱스를 구합니다. (가장 어두울 땐 공백, 밝을 수록 우측 문자)

### 2.4 Google Material 3 기반 컬러 팔레트 시스템
모든 컬러는 루미넌스 `val` 값(0.0 ~ 1.0)을 구간별 분할하여 보간(Lerp)합니다. 
가장 피크(Peak) 부분은 무조건 **순수 파랑(`#0000FF`)**에 수렴해야 합니다.
 
* **구간 1 (0.0 ~ 0.25):** Background -> Color 1
* **구간 2 (0.25 ~ 0.5):** Color 1 -> Color 2
* **구간 3 (0.5 ~ 0.75):** Color 2 -> Color 3
* **구간 4 (0.75 ~ 0.9):** Color 3 -> Peak (`#0000FF`)
* **구간 5 (0.9 ~ 1.0):** Peak -> Glow (`#5050FF`)

현재 세 가지 변형 팔레트가 정의되어 있습니다 (모두 시안/청록 배제):
1. **Main:** `[#010204]` -> `[#08081a]` -> `[#0d0d3a]` -> `[#1a1a6e]` -> `[#0000FF]`
2. **Navy/Cobalt:** 묵직한 네이비 기반 (`#020518` 중심)
3. **Indigo:** 미세한 보라빛이 도는 우주 색 (`#0a0418` 중심)

### 2.5 후처리 및 최적화 (Post-Processing)
* CRT 텍스처, 가로줄 스캔라인(Scanline), 모서리 비네팅(Vignette) 효과는 캔버스 렌더링이 아닌 CSS Overlay로 분리하여 퍼포먼스를 극히 끌어올렸습니다.
* 블루 글로우(Bloom) 효과는 CSS의 `drop-shadow` 속성을 통해 캔버스 통째로 필터링했습니다.

## 3. 다음 작업자(에이전트)를 위한 가이드라인: Phase 3 (After Effects 스크립팅)

다음 에이전트는 사용자가 지시를 내리면, 위에서 정의한 시스템 명세를 **After Effects의 `ExtendScript (.jsx)`** 문법으로 포팅(Porting)해야 합니다.

1. ** sampleImage 표현식 적용:** AE에서는 픽셀을 순회하는 웹 방식이 아니라, 텍스트 레이어로 만들어진 그리드 시스템에 Expression(표현식)을 주사하여 `sampleImage`로 하단 비디오 레이어의 루미넌스를 측정합니다.
2. **극단적 성능 주의 (퍼포먼스 스파이크):** 1920x1080 캠프에서 12px 폰트로 레이어를 만들면 약 14,400개의 텍스트 레이어가 생성되며 AE 애플리케이션이 크래시(다운)될 수 있습니다. 
3. **대안 검토:** 다음 에이전트는 레이어를 무식하게 깔기 전에 **"프리컴프 해상도 극한 다운스케일 + Effect 조합 플로"**(과거 지침서 경로 2번/3번 방식) 또는 AE의 퍼포먼스를 해치지 않는 선상(예: 셀 크기 30~48px로 타협 후 생성)에 대해 논리적으로 먼저 제안해야 합니다.
4. 웹캠이 아닌 지정된 영상(`choi.mp4`) 소스를 바탕으로 작업하게 될 확률이 높습니다.
